Definition Cheat Sheet

Git Definitions

In these definitions, the usual part of speech of the word is shown. As is often the case with developer terminology, most of these terms can be nouns or verbs depending on the circumstance. I've listed the most common use that I've seen. If a word is commonly used as both a noun and a verb, I've denoted the term with both.

These are my definitions - you can find official Git documentation here git-scm.com.

git

noun

A revision control software installed on your computer.

github

noun

A social media site for sharing projects that are revision controlled with Git.

repository

noun

A folder with files that are revision controlled with Git.

History

noun

The record of changes in a repo.

commit

noun, verb

A commit is how you tell Git to record changes in the history.

branch

noun, verb

A branch is a separate history. When you create a new branch, Git will keep track of your commits on that branch separately from other branches.

master branch

noun

The version history in a repo that has approved changes. You can easily switch between the master and other branches.

merge

verb

Combine the files and history from one branch into another branch.

remote

adjective

Describes files/changes on GitHub or some other server (as opposed to local).

local

adjective

Describes files/changes on your computer (as opposed to remote).

clone

verb

To download all the files (and the history of those files) from a remote repo onto your local computer.

push

verb

Add committed changes from the repo on your computer to GitHub.

pull

verb

Bring changes from GitHub to your local repo.

pull request

noun

This is a formalized way of asking for your changes to be approved. You create a pull request to give collaborators an easy way to view exactly what changes you made and state whether they approve of the changes or not.
